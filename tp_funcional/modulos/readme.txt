Para correr los tests es necesario tener todos los módulos presentes
en este .zip junto con todos los módulos del TP.
Si tienen algunas funciones implementadas pero no todas pueden comentar
los tests tanto en el conjunto de tests al final del archivo Test.hs como
sus respectivas definiciones que llevan el mismo nombre.
Para correr los tests deberán abrir el archivo Test.hs con el Hugs y correr
la función main.
En caso de que un test falle se indicará qué test falló, cuál era el resultado
esperado, y cuál fue el resultado obtenido. Pueden ver el archivo Test.hs para
ver cómo están armados los tests y qué debería dar cada uno.
Es condición necesaria (pero no suficiente) para aprobar el TP pasar los tests.
